Pending

Overview

  • Pending is a calendar-based task manager web application. It is a multi-user platform, where a user can create an account and new tasks. Tasks can be checked off, edited or deleted. These are displayed in a weekly calendar view, and it keeps track of how many tasks are still pending for the week which runs Monday through Sunday.

Post-Mortem

  • I am very satisfied with the outcome of the project; it is an application that I could see myself continue to work on after this class in order to keep improving it and making it into something I can use daily to replace my handwritten list. The beginning of the process was a little bumpy because I was trying to wrap my head around MVC. It was helpful to use Domomaker as starter files, and build my app off of that. I think I can improve efficiency, and better error checking (for instance, if an invalid date is entered on the URL). If I were to do this again, I would create a functional document with all the requirements and functionality I hope to implement in order to stay focused on the scope, rather than try to add pieces here and there when they come to mind.

    In a previous class, I worked on a similar team project using Django except that I only created templates for the MTV architecture. This time around, I really enjoyed having the ability to work more on the backend and understand how all the parts and pieces come together.
